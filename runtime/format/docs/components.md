# Components added to the scene object

Components are added to all objects that accept one. These are put in the `components` array of the object, and they are defined as objects with a `type` property that specifies the type of the component, and other properties that define the values for that component.

## Script (`type = "script"`)

A script component is a component that can be added to an object to make it execute a script, which can be used to define custom behavior and logic for the object in the scene. It has the following properties:

* `source`: The source of the script, which is a reference to a script file (`.ts`) that defines the code that will be executed by the script component. This can be used to create custom behavior for the object in the scene by writing code in the specified script file.
* `variables`: A list of pairs of `name` and `value`, which are meant to be used as variables that can be accessed and modified by the script code. The `name` is a string representing the name of the variable, and the `value` can be any JSON value (e.g., string, number, boolean, array, object, etc.) that represents the initial value of the variable. These variables can be used in the script code to store and manipulate data related to the object in the scene. These values though, have to start with a type definer (e.g., `str<Hello, world!>`, `num<42>`, `bool<true>`, `ref<myObject>`, etc.) to indicate the type of the variable, since the script code will need to know the type of the variable to be able to use it correctly.

## Rigidbody (`type = "rigidbody"`)

A rigidbody component is a component that can be added to an object to make it affected by physics. It has the following properties:

* `mass`: The mass of the rigidbody, defined as a single number representing the mass in units. A higher value means that the object will be heavier and will be affected more by gravity and other forces.
* `sendSignal`: The signal that the rigidbody will send when it collides with another object, defined as a string representing the name of the signal. This can be used to trigger events or other actions in the scene when the object collides with something.
* `isSensor`: A boolean value that indicates whether the rigidbody is a sensor or not. If `true`, the rigidbody will not be affected by physics and will not collide with other objects, but it will still send signals when it overlaps with other objects. If `false`, the rigidbody will be affected by physics and will collide with other objects as normal.
* `collider`: An object that defines the collider shape of the rigidbody, which can be used to define the shape and size of the object for collision detection. The collider can be defined as an object with a `type` property that specifies the type of the collider (e.g., `box`, `sphere`, etc.), and other properties that define the values for that collider (e.g., `size`, `radius`, etc.).
* `friction`: The friction of the rigidbody, defined as a single number representing the coefficient of friction (values between 0 and 1). A higher value means that the object will be more resistant to sliding against other objects.
* `tags`: An array of strings representing the tags that are associated with the rigidbody, which can be used to categorize and filter objects in the scene based on their tags. This can be useful for things like collision filtering, where you only want certain objects to collide with each other based on their tags.
* `damping`: The damping of the rigidbody, with two values (`linear` and `angular`) representing the linear and angular damping of the rigidbody, defined as a single number representing the damping in units per second. A higher value means that the object will slow down faster when it is moving or rotating.
* `restituition`: The restitution of the rigidbody, defined as a single number representing the coefficient of restitution (values between 0 and 1). A higher value means that the object will be bouncier and will retain more of its velocity after colliding with another object.
* `motionType`: The motion type of the rigidbody, defined as a string that can be either `static`, `dynamic` or `kinematic`. A `static` rigidbody will not move and will not be affected by physics, but it can still collide with other objects. A `dynamic` rigidbody will be affected by physics and will collide with other objects as normal. A `kinematic` rigidbody will not be affected by physics and will not collide with other objects, but it can still be moved manually by setting its position and rotation in the scene.

## AudioPlayer (`type = "audio_player"`)

An audio player component is a component that can be added to an object to make it play audio. It has the following properties:

* `source`: The source of the audio, which is a reference to an audio file (e.g., `.mp3`, `.wav`, etc.) that defines the sound that will be played by the audio player.
* `position`: The position of the audio player in the scene, defined as an array of three numbers representing the x, y and z coordinates. This can be used to create spatial audio effects, where the sound will be louder when the listener is closer to the audio player and quieter when the listener is farther away.
* `useSpatialization`: A boolean value that indicates whether to use spatialization for the audio or not. If `true`, the audio will be spatialized based on the position of the audio player and the listener in the scene. If `false`, the audio will be played as a non-spatialized sound, which means that it will be heard at the same volume regardless of the position of the audio player and the listener in the scene.

## Joint (`type = "joint"`)

A joint component is a component that can be added to an object to connect it to another object with a joint. It has the following properties:

* `parent`: The GameObject that is the parent of the joint, which is the object that the joint will be attached to. This is defined as a reference to another object in the scene (e.g., by its ID or name).
* `child`: The GameObject that is the child of the joint, which is the object that will be connected to the parent object with the joint. This is defined as a reference to another object in the scene (e.g., by its ID or name).
* `space`: The space in which the joint will operate, defined as a string that can be either `world` or `local`. If `world`, the joint will operate in world space, which means that the position and rotation of the joint will be defined relative to the world origin. If `local`, the joint will operate in local space, which means that the position and rotation of the joint will be defined relative to the parent object.
* `anchor`: The anchor point of the joint, defined as an array of three numbers representing the x, y and z coordinates of the anchor point in the space defined by the `space` property. This is the point where the joint will be attached to the parent and child objects. 
* `breakForce`: The break force of the joint, defined as a single number representing the force in units that will cause the joint to break. A higher value means that the joint will be stronger and will require more force to break.
* `breakTorque`: The break torque of the joint, defined as a single number representing the torque in units that will cause the joint to break. A higher value means that the joint will be stronger and will require more torque to break.

## FixedJoint (`type = "fixed_joint"`)
A fixed joint is a type of joint that can be added to an object to connect it to another object with a fixed joint, which means that the two objects will be rigidly connected and will not be able to move relative to each other. It has the same properties as the `Joint` component, but with the `type` property set to `fixed_joint` instead of `joint`.

## HingeJoint (`type = "hinge_joint"`)
A hinge joint is a type of joint that can be added to an object to connect it to another object with a hinge joint, which means that the two objects will be connected in a way that allows them to rotate around a single axis relative to each other. It has the same properties as the `Joint` component, but with the `type` property set to `hinge_joint` instead of `joint` instead. Additionally, it has the following properties:

* `axis1`: The first axis of the hinge joint, defined as an array of three numbers representing the x, y and z components of the first axis vector. This is the axis around which the child object will rotate relative to the parent object.
* `axis2`: The second axis of the hinge joint, defined as an array of three numbers representing the x, y and z components of the second axis vector. This is the axis that is perpendicular to the first axis and defines the plane in which the child object will rotate relative to the parent object.
* `limits`: Angular limits with three properties:
  * `isEnabled`: A boolean value that indicates whether the angular limits are enabled or not. If `true`, the child object will be limited in its rotation relative to the parent object based on the `min` and `max` properties. If `false`, the child object will be able to rotate freely around the hinge joint without any limits.
  * `minAngle`: The minimum angle of the hinge joint, defined as a single number representing the minimum angle in degrees that the child object can rotate relative to the parent object when the limits are enabled. A higher value means that the child object will be able to rotate more in the negative direction relative to the parent object.
  * `maxAngle`: The maximum angle of the hinge joint, defined as a single number representing the maximum angle in degrees that the child object can rotate relative to the parent object when the limits are enabled. A higher value means that the child object will be able to rotate more in the positive direction relative to the parent object.
* `motor`: Motor settings with three properties:
  * `isEnabled`: A boolean value that indicates whether the motor is enabled or not. If `true`, the motor will apply a torque to the child object to try to reach the target velocity defined by the `targetVelocity` property. If `false`, the motor will not apply any torque to the child object.
  * `targetVelocity`: The target velocity of the motor, defined as a single number representing the target angular velocity in degrees per second that the motor will try to achieve for the child object when the motor is enabled. A higher value means that the motor will try to rotate the child object faster around the hinge joint.
  * `maxTorque`: The maximum torque of the motor, defined as a single number representing the maximum torque in units that the motor can apply to the child object when trying to reach the target velocity. A higher value means that the motor will be able to apply more torque to try to reach the target velocity, which can be useful for overcoming resistance or inertia in the system.

## SpringJoint (`type = "spring_joint"`)
A spring joint is a type of joint that can be added to an object to connect it to another object with a spring joint, which means that the two objects will be connected in a way that allows them to move relative to each other while being pulled together by a spring force. It has the same properties as the `Joint` component, but with the `type` property set to `spring_joint` instead of `joint` instead. Additionally, it has the following properties:

* `anchorB`: The anchor point of the spring joint on the child object, defined as an array of three numbers representing the x, y and z coordinates of the anchor point in the local space of the child object. This is the point where the spring force will be applied to the child object.
* `restLength`: The rest length of the spring joint, defined as a single number representing the length in units that the spring will try to maintain between the parent and child objects. A higher value means that the spring will try to keep the objects farther apart, while a lower value means that the spring will try to keep the objects closer together.
* `useLimits`: A boolean value that indicates whether to use limits for the spring joint or not. If `true`, the spring joint will only apply force when the distance between the parent and child objects is outside of the range defined by the `minLength` and `maxLength` properties. If `false`, the spring joint will apply force based on the difference between the current length and the rest length regardless of any limits.
* `minLength`: The minimum length of the spring joint, defined as a single number representing the minimum length in units that the spring will allow between the parent and child objects when limits are enabled. A higher value means that the spring will not allow the objects to get too close together.
* `maxLength`: The maximum length of the spring joint, defined as a single number representing the maximum length in units that the spring will allow between the parent and child objects when limits are enabled. A higher value means that the spring will not allow the objects to get too far apart.
* `spring`: The spring force of the spring joint with properties:
  * `enabled`: A boolean value that indicates whether the spring force is enabled or not. If `true`, the spring joint will apply a force based on Hooke's law to try to maintain the rest length between the parent and child objects. If `false`, the spring joint will not apply any force based on Hooke's law.
  * `mode`: The mode of the spring force, either `frequencyAndDamping` or `stiffnessAndDamping`. If `frequencyAndDamping`, the spring force will be defined by the `frequency` and `damping` properties. If `stiffnessAndDamping`, the spring force will be defined by the `stiffness` and `damping` properties.
  * `frequencyHz`: The frequency of the spring force when using `frequencyAndDamping` mode, defined as a single number representing the frequency in hertz. A higher value means that the spring will oscillate faster.
  * `damping`: The damping of the spring force, defined as a single number representing the damping in units per second. A higher value means that the spring will lose energy faster and will come to rest more quickly.
  * `stiffness`: The stiffness of the spring force when using `stiffnessAndDamping` mode, defined as a single number representing the stiffness in units per length. A higher value means that the spring will be stronger and will try harder to maintain the rest length between the parent and child objects.
  * `damping`: The damping of the spring force, defined as a single number representing the damping in units per second. A higher value means that the spring will lose energy faster and will come to rest more quickly.
